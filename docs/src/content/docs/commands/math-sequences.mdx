# Math & Sequences

`seq` for generating numeric sequences and `expr` for arithmetic and string expression evaluation.

---

## seq

Generate a sequence of numbers, one per line or joined with a separator.

```
seq [OPTIONS] LAST
seq [OPTIONS] FIRST LAST
seq [OPTIONS] FIRST INCREMENT LAST
```

### Flags

| Flag | Description |
|------|-------------|
| `-s SEP`, `--separator=SEP` | Use SEP as the separator between numbers (outputs a single `BashText` string instead of individual objects) |
| `-w`, `--equal-width` | Pad numbers with leading zeros so all have equal width |

### SeqOutput Properties

Without `-s`, each result is a `PsBash.SeqOutput` object:

| Property | Type | Description |
|----------|------|-------------|
| `Value` | long or double | The numeric value of the sequence element |
| `Index` | int | Zero-based position in the sequence |
| `BashText` | string | Formatted string representation (includes padding when `-w` is used) |

<Aside type="note">
When `-s` is used, seq returns a single `BashText` object with all values joined by the separator instead of individual `SeqOutput` objects.
</Aside>

### Examples

**Count from 1 to 5:**

```powershell
seq 5
```

```text
1
2
3
4
5
```

Each line is a typed `SeqOutput` object:

```powershell
$nums = seq 5
$nums[2].Value   # 3
$nums[2].Index   # 2
```

**Specify a start and end:**

```powershell
seq 3 7
```

```text
3
4
5
6
7
```

**Custom increment:**

```powershell
seq 0 2 10
```

```text
0
2
4
6
8
10
```

**Countdown with negative increment:**

```powershell
seq 5 -1 1
```

```text
5
4
3
2
1
```

**Comma-separated output:**

```powershell
seq -s ', ' 1 5
```

```text
1, 2, 3, 4, 5
```

**Equal-width zero padding:**

```powershell
seq -w 1 100
```

```text
001
002
003
...
099
100
```

**Decimal sequences:**

```powershell
seq 0.0 0.5 2.0
```

```text
0.0
0.5
1.0
1.5
2.0
```

**Pipeline composition -- types survive through the pipeline:**

```powershell
seq 1 10 | grep '[13579]$'
```

```text
1
3
5
7
9
```

## expr

Evaluate arithmetic, comparison, or string expressions and return the result.

```
expr EXPRESSION
```

Expressions use infix notation for arithmetic and comparisons, and keyword-prefix notation for string operations.

### Arithmetic Operators

| Operator | Description |
|----------|-------------|
| `+` | Addition |
| `-` | Subtraction |
| `*` | Multiplication |
| `/` | Integer division (truncates toward zero) |
| `%` | Modulus (remainder) |

### Comparison Operators

| Operator | Description |
|----------|-------------|
| `=` | Equal (case-sensitive for strings) |
| `!=` | Not equal |
| `<` | Less than |
| `<=` | Less than or equal |
| `>` | Greater than |
| `>=` | Greater than or equal |

Comparisons return `1` for true and `0` for false. When both operands are integers, numeric comparison is used; otherwise string comparison applies.

### String Operations

| Syntax | Description |
|--------|-------------|
| `expr length STRING` | Length of the string |
| `expr substr STRING POS LEN` | Substring starting at 1-based POS for LEN characters |
| `expr index STRING CHARS` | 1-based position of the first character in CHARS found in STRING, or 0 if none |
| `expr match STRING REGEX` | Match STRING against a POSIX BRE anchored at start; returns the captured group if `\(...\)` is present, otherwise the match length |

### ExprOutput Properties

Each result is a `PsBash.ExprOutput` object:

| Property | Type | Description |
|----------|------|-------------|
| `Value` | long or string | The result as a number (when the result is numeric) or string |
| `BashText` | string | String representation of the result |

### Examples

**Basic arithmetic:**

```powershell
expr 2 + 3
```

```text
5
```

The result is a typed object:

```powershell
$r = expr 10 '*' 5
$r.Value   # 50
```

<Aside type="tip">
Wrap `*` in quotes to prevent PowerShell from expanding it as a glob. The same applies to `<` and `>` which PowerShell interprets as redirection operators.
</Aside>

**Integer division and modulus:**

```powershell
expr 17 / 5
```

```text
3
```

```powershell
expr 17 '%' 5
```

```text
2
```

**Comparison:**

```powershell
expr 10 '>' 5
```

```text
1
```

```powershell
expr 3 '=' 3
```

```text
1
```

**String length:**

```powershell
expr length 'hello world'
```

```text
11
```

**Substring extraction:**

```powershell
expr substr 'hello world' 7 5
```

```text
world
```

**Character index:**

```powershell
expr index 'hello' 'lo'
```

```text
3
```

Returns the 1-based position of the first occurrence of any character from the search string. Here `l` appears at position 3.

**Regex match with capture group:**

```powershell
expr match 'archive-2024.tar.gz' '.*-\([0-9]*\)'
```

```text
2024
```

The `\(...\)` capture group extracts the year. Without a capture group, match returns the length of the matched portion.

**Single operand passthrough:**

```powershell
expr 'hello'
```

```text
hello
```

When given a single operand, expr echoes it as the result.

### seq vs expr

| | `seq` | `expr` |
|---|-------|--------|
| Purpose | Generate number sequences | Evaluate expressions |
| Output type | `SeqOutput` per number | Single `ExprOutput` |
| Decimal support | Yes | Integer arithmetic only |
| String operations | No | `length`, `substr`, `index`, `match` |
| Pipeline use | Generates multiple objects for piping | Returns a single result |
