## DIRECTIVE 1. ORACLE FIRST. ASSERT LAST.

RULE: HAND-WRITTEN ASSERT MEANS AUTHOR'S MENTAL MODEL. AUTHOR'S MENTAL MODEL MAKES BUG.
RULE: BASH IS ORACLE. PS-BASH IS SUSPECT.

PROCEDURE:
- WRITE SCRIPT.
- RUN IN REAL `bash`. CAPTURE STDOUT. STDERR. EXIT.
- RUN IN PS-BASH. CAPTURE STDOUT. STDERR. EXIT.
- DIFF BYTES. SAME = PASS. DIFFERENT = FAIL.
- IF BASH NOT AVAILABLE ON PLATFORM, GOLDEN FILE FROZEN FROM LINUX RUN.

USE HAND-WRITTEN ASSERT ONLY WHEN:
- BEHAVIOR IS PS-BASH-SPECIFIC (e.g. `Invoke-Bash*` cmdlet surface, sqlite history file).
- BASH HAS NO EQUIVALENT (e.g. PowerShell pipeline object preservation).
- DOCUMENT WHY IN TEST COMMENT. ONE LINE.

CANONICALIZE BEFORE DIFF:
- STRIP ANSI ESCAPES.
- NORMALIZE CRLF -> LF.
- STRIP TRAILING WHITESPACE PER LINE.
- TRAILING NEWLINE: PRESERVE. IT MATTERS.
- DO NOT NORMALIZE CASE. DO NOT NORMALIZE PATH SEPARATORS UNLESS TEST SAYS PATH-AGNOSTIC.

## DIRECTIVE 2. NUMBERS OR IT DID NOT HAPPEN.

NO PARITY CLAIM WITHOUT NUMBERS. WORDS LIE. NUMBERS DO NOT.

BARS:
- BRANCH COVERAGE >= 80% ON `PsEmitter.cs`.
- BRANCH COVERAGE >= 80% ON `BashParser.cs`.
- BRANCH COVERAGE >= 70% ON `PsBash.psm1` (per `Invoke-Bash*` function).
- DIFF COVERAGE >= 90% ON EVERY PR THAT TOUCHES src/PsBash.Core OR src/PsBash.Module.
- MUTATION SCORE >= 60% ON RUNTIME (Stryker.NET; deferred until Phase 0 lands).
- DIFFERENTIAL CASES >= 5 PER FEATURE IN THE FAILURE-SURFACE MATRIX.

FLAKE BAR:
- ZERO FLAKES OVER 100 RE-RUNS ON CI MATRIX BEFORE MERGE.
- ONE FLAKE = QUARANTINE TAG. NOT DISABLE. NOT IGNORE.
- QUARANTINE > 7 DAYS = OPEN BUG. ASSIGN. FIX OR DELETE TEST.

## DIRECTIVE 3. FAILURE-SURFACE MATRIX, NOT GRAMMAR LIST.

OLD WAY: LIST LANGUAGE FEATURES. CHECK BOX. SHIP. BUG.
NEW WAY: LIST FAILURE SURFACES. PROBE EACH. SHIP. NO BUG.

EVERY FEATURE GETS A 2D PROBE: FEATURE x FAILURE-AXIS.

FAILURE AXES (REQUIRED PER FEATURE):
1. EMPTY INPUT.
2. LARGE INPUT (>= 10 MB STREAMED THROUGH PIPE).
3. UNICODE INPUT (BOM, NON-ASCII, EMOJI, COMBINING MARKS).
4. CRLF INPUT.
5. BROKEN PIPE (READER CLOSES EARLY).
6. SLOW READER (CONSUMER BLOCKS; PRODUCER MUST NOT BUFFER UNBOUNDED).
7. SIGNAL DURING EXECUTE (CTRL-C MID-PIPE).
8. EXIT CODE PROPAGATION (LAST CMD; PIPELINE WITH PIPEFAIL; NEGATED `!`).
9. STDERR INTERLEAVE (`2>&1`, `|&`).
10. WORKING DIR STATE (cd PERSIST? RESET BETWEEN COMMANDS?).
11. ENVIRONMENT LEAK (DOES `VAR=x cmd` LEAK VAR?).
12. QUOTING / INJECTION (UNQUOTED VAR WITH SPACES, IFS, BACKTICKS, `;`).
13. PLATFORM-LOCKED FILE (WINDOWS BINARY IN-USE).
14. MISSING TARGET (FILE, CMD, FUNCTION DOES NOT EXIST).
15. RECURSION DEPTH (FUNCTIONS, COMMAND SUB, EVAL).

NOT EVERY AXIS APPLIES. JUSTIFY SKIP IN AUDIT DOC. NO SKIP WITHOUT REASON.

## DIRECTIVE 4. MODE INTERACTION MATRIX. NOT MODE PARITY.

MODES:
- M1: `-c` (one-shot string).
- M2: STDIN PIPE (`echo cmd | ps-bash`).
- M3: FILE ARG (`ps-bash script.sh`).
- M4: INTERACTIVE TTY (REPL).
- M5: `Invoke-BashEval` CMDLET (in-process).
- M6: `Invoke-BashSource` CMDLET (in-process).

EACH FEATURE GETS A MODE-AXIS PROBE. NOT JUST "DOES IT RUN".

MODE-INTERACTION AXES:
A. TTY VS NO-TTY (BUFFERING DIFFERS).
B. LINE BUFFER VS BLOCK BUFFER (PROGRESS BARS, REAL-TIME OUTPUT).
C. SIGNAL DELIVERY (CTRL-C TO PIPELINE; SIGTERM TO BG JOB).
D. PROMPT REDRAW AFTER ASYNC OUTPUT (M4 ONLY).
E. HISTORY PERSISTENCE (M4 ONLY).
F. PROFILE LOAD (M4 + M3 IF SHEBANG).
G. EXIT CODE BACK TO PARENT.
H. STDIN CLOSED MID-READ.

PER FEATURE PER MODE: AT LEAST ONE INTERACTION-AXIS PROBE. JUSTIFY OMIT.

## DIRECTIVE 5. PLATFORM MATRIX. ALL THREE OR NOTHING.

PS-BASH RUNS WIN/LINUX/MAC. CI MUST RUN ALL THREE. GREEN ON ONE != GREEN.

PLATFORM-SPECIFIC RISK ZONES (KNOWN BAD):
- WINDOWS: NO SIGHUP. LOCKED BINARY. CRLF. CONHOST VS WINDOWS TERMINAL. PATH SEPARATOR. CASE-INSENSITIVE FS.
- LINUX: PROC FS LEAKS. `/dev/null` PERMISSIONS.
- MAC: BSD `sed`/`date` FLAGS. APPLE-SIGNED PWSH SANDBOX.

EVERY NEW TEST FILE DECLARES PLATFORM SUPPORT IN TRAIT/SKIP.
PLATFORM-LOCKED TESTS USE `[Trait("Platform","Windows")]` / `[SkippableFact]`.
NO TEST SILENTLY NO-OP ON A PLATFORM. SKIP MUST EMIT REASON.

## DIRECTIVE 6. DETERMINISM HARNESS. NO FLAKE.

INTERACTIVE TESTS USE PTY. PTY = TIMING. TIMING = FLAKE. UNLESS:

INTERACTIVE TEST FIXTURE MUST:
- FIX `TERM=xterm-256color`.
- FIX `LANG=C.UTF-8`.
- FIX `COLUMNS=120 LINES=40`.
- FIX `PROMPT_COMMAND=` (empty), `PS1='PSBASH$ '`, `PS2='> '` (so prompt is byte-stable).
- ISOLATE `HOME` TO TEST TEMP DIR.
- ISOLATE `PSBASH_HISTORY` TO TEST TEMP FILE.
- DISABLE `.psbashrc` UNLESS TEST EXPLICITLY OPTS IN.
- WAIT-FOR-PROMPT, NEVER `Sleep(N)`. PROMPT REGEX = ONE LINE. TIMEOUT = 5 s.
- ON TIMEOUT: DUMP TRANSCRIPT, ENV, CWD, EXIT CODE, TRANSPILED PS, RUNTIME STDERR.

NO `Thread.Sleep`. NO `await Task.Delay`. ANY SLEEP = REJECT IN REVIEW.

## DIRECTIVE 7. NEGATIVE TESTS ARE PRIMARY. NOT BONUS.

EVERY FEATURE TASK MUST INCLUDE NEGATIVE CASES:
- MISSING FILE.
- BROKEN PIPE.
- DISK FULL (SIMULATED VIA FAILING WRITE STREAM, NOT REAL DISK).
- CHILD PROCESS CRASH (EXIT 139, SIGSEGV).
- PROFILE SCRIPT THROWS.
- ALIAS LOOP (`alias a=b; alias b=a; a`).
- COMMAND-NAME WITH SPACES.
- UNQUOTED VAR WITH GLOB CHARS.
- IFS MUTATION.

POSITIVE-ONLY TEST SUITE = REJECT IN REVIEW.

## DIRECTIVE 8. CROSS-MODE CANARY. EVERY PR.

CANARY SUITE = ~30 TESTS. RUNS IN ALL 6 MODES ON EVERY PR.
ONE TEST PER FEATURE IN THE FAILURE-SURFACE MATRIX. SMALLEST POSSIBLE.
PURPOSE: CATCH MODE-ONLY REGRESSIONS BEFORE MERGE. NOT EXHAUSTIVE.
LOCATED: `src/PsBash.Canary.Tests/`.
RUNS IN UNDER 60 SECONDS PER MODE PER PLATFORM. HARD CAP.

## DIRECTIVE 9. OBSERVABILITY ON FAILURE. ALWAYS.

WHEN A TEST FAILS, CI ARTIFACT MUST CONTAIN:
- INPUT SCRIPT (FILE + LINE).
- TRANSPILED POWERSHELL OUTPUT.
- STDOUT.
- STDERR.
- EXIT CODE.
- WORKING DIR AT TIME OF FAILURE.
- ENV SNAPSHOT (FILTERED: NO SECRETS).
- PLATFORM, OS VERSION, PWSH VERSION.
- FOR PTY: FULL TRANSCRIPT WITH TIMING.

NO "EXPECTED X GOT Y" ALONE. NO REPRO INSTRUCTION ALONE. ARTIFACT BUNDLE OR NO BUG REPORT.

## DIRECTIVE 11. ACCEPTANCE BARS. MEASURABLE OR REJECTED.

EVERY PHASE TASK MUST DEFINE:
- INPUT: WHAT MUST EXIST BEFORE START.
- OUTPUT: FILE/TEST/NUMBER PROVING DONE.
- BAR: NUMERIC THRESHOLD (COVERAGE %, RUN COUNT, DIFFERENTIAL CASE COUNT).
- VERIFICATION: COMMAND A REVIEWER RUNS TO CONFIRM.

NO "FILL GAPS". NO "IMPROVE COVERAGE". NO "ADD TESTS".
USE: "ADD >= 5 DIFFERENTIAL CASES PER FAILURE AXIS FOR FEATURE X. RAISE BRANCH COVERAGE ON EMITTER FOR X TO >= 80%. CMD: `./scripts/test.sh --filter X --coverage`."

## DIRECTIVE 12. SECURITY PROBES. NOT OPTIONAL.

TRANSPILER LEAKS QUOTING AT THE BASH/POWERSHELL SEAM. SHIP-STOPPER.

REQUIRED SECURITY CASES PER FEATURE THAT TOUCHES VAR EXPANSION OR EXEC:
- VAR CONTAINING `;` (cmd injection at PS layer).
- VAR CONTAINING `$(...)` (nested expansion).
- VAR CONTAINING POWERSHELL SCRIPTBLOCK CHARS `{}` `$(`.
- COMMAND NAME FROM USER INPUT (alias, function, source target).
- IFS=$'\n' WITH GLOB.
- HEREDOC CONTAINING `"` AND `$` AND BACKTICK.

ASSERT: TRANSPILED OUTPUT DOES NOT EXECUTE INJECTION PAYLOAD.

## DIRECTIVE 13. KNOWN-BAD MEMORY. RESPECT IT.

BUGS HISTORY IS INPUT. SEE @MEMORY.md AND `docs/solutions/`.
KNOWN BAD CATEGORIES (DO NOT REGRESS):
- WINDOWS PROCESS DEATH (NO SIGHUP).
- PROCESS SPAWN WITHOUT TIMEOUT + KILL-TREE (LOCKUP).
- UNQUOTED `|` IN xunit `--filter` (TEST RUN HANG).
- LASTEXITCODE SAVE/RESTORE COLLISIONS.
- ERR TRAP ON STALE LASTEXITCODE.
- FLAKY MACOS `Invoke-BashSource` TESTS.

EVERY KNOWN-BAD = ONE PERMANENT REGRESSION TEST IN CANARY OR ESCALATION SUITE.
